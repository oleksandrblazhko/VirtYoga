# Сценарії використання матриць для опису суглобів людини у "Віртуальній Йозі"

## Вступ

Цей документ пропонує кілька способів застосування концепцій з лекції з лінійної алгебри (зокрема, матриць) до ІТ-проєкту "Віртуальна Йога". Мета — надати студентам конкретні приклади того, як абстрактні математичні поняття використовуються на практиці, роблячи процес навчання більш захоплюючим та відповідним їхнім інтересам.

Додаток "Віртуальна Йога" відстежує скелет користувача, що складається з 33 суглобів, кожен з яких має 3D-координати (x, y, z). Ми можемо використовувати матриці для представлення та маніпулювання цими даними.

---

## Сценарій 1: Представлення суглобів та скелета

### Поняття з лекції:
*   Означення матриці
*   Матриця-рядок та матриця-стовпець

### Застосування до проєкту:

1.  **Представлення одного суглоба:**
    Кожен суглоб має три координати (x, y, z). Ми можемо представити один суглоб як **матрицю-рядок** (матриця розміром 1x3) або **матрицю-стовпець** (матриця розміром 3x1).

    Наприклад, положення `лівого плеча` (суглоб 11) можна представити так:
    `Суглоб_11 = [x_11, y_11, z_11]`

2.  **Представлення всього скелета:**
    Весь скелет складається з 33 суглобів. Ми можемо представити повну позу людини як одну матрицю розміром 33x3. Кожен рядок у цій матриці відповідатиме певному суглобу, а три стовпці представлятимуть координати x, y та z цього суглоба.

    `Поза_Скелета =`
    ```
    | x_0,  y_0,  z_0  |  (ніс)
    | x_1,  y_1,  z_1  |  (внутрішня частина лівого ока)
    | ...   ...   ...  |
    | x_32, y_32, z_32 |  (вказівний палець правої ноги)
    ```
    Ця матриця `Поза_Скелета` надає повний знімок пози людини в певний момент часу.

3.  **Зберігання еталонних поз:**
    "Правильні" пози йоги, які додаток використовує для порівняння, також можна зберігати у вигляді матриць 33x3 у базі даних. Наприклад, поза "Воїн II" буде зберігатися як матриця `Поза_Воїн_II`.

---

## Сценарій 2: Переміщення пози (Трансляція)

### Поняття з лекції:
*   Додавання матриць

### Застосування до проєкту:

Уявіть, що користувач виконує ідеальну позу йоги, але знаходиться не в центрі огляду камери. Його поза правильна, але положення зміщене. Додаток може захотіти змістити еталонну позу, щоб вона відповідала загальному положенню користувача. Це є **трансляція** (або перенесення).

Ми можемо визначити вектор трансляції (у вигляді матриці 1x3), який представляє необхідне зміщення вздовж осей x, y та z.
`Вектор_Трансляції = [delta_x, delta_y, delta_z]`

Щоб перемістити всю еталонну позу, ми можемо додати цей вектор трансляції до **кожного рядка** матриці еталонної пози.

`Нова_Поза = Стара_Поза + Матриця_Трансляції`

Де `Матриця_Трансляції` — це матриця 33x3, у якій кожен рядок є `Вектором_Трансляції`.

**Приклад:**
Якщо ми хочемо змістити всю позу на 10 одиниць по осі X, ми б додали матрицю:
```
| 10, 0, 0 |
| 10, 0, 0 |
| ...      |
| 10, 0, 0 |
```
до матриці `Поза_Скелета`. Це використовує принцип **додавання матриць**.

## Сценарій 3: Масштабування пози

### Поняття з лекції:
*   Множення матриці на число (скалярне множення)

### Застосування до проєкту:

Еталонна поза йоги в базі даних може бути розрахована для людини середнього зросту. Щоб зробити порівняння справедливим для вищого або нижчого користувача, додатку може знадобитися масштабувати еталонну позу.

Ми можемо зробити це, помноживши всю матрицю еталонної пози на одне число (скаляр).

`Масштабована_Поза = k * Еталонна_Поза`

*   Якщо `k > 1`, поза стає більшою (для вищої людини).
*   Якщо `0 < k < 1`, поза стає меншою (для нижчої людини).

Це є прямим застосуванням **скалярного множення**, де кожен елемент матриці множиться на скаляр `k`.

---

## Сценарій 4: Обертання кінцівок (Вступ до обертань)

### Поняття з лекції:
*   Множення матриць

### Застосування до проєкту:

Це більш просунуте застосування, але це потужна демонстрація **множення матриць**. У 3D-графіці та біомеханіці обертання виконуються за допомогою матриць обертання.

Наприклад, щоб перевірити кут передпліччя користувача відносно плеча (згин ліктя), додатку потрібно розуміти орієнтацію, а не лише положення.

1.  Сегмент кінцівки (наприклад, передпліччя, від ліктя до зап'ястя) можна представити як вектор.
2.  Щоб обертати цю кінцівку в 3D-просторі, ми **множимо її вектор на матрицю обертання**.

Для обертання навколо осі Z на кут `θ` матриця обертання має вигляд:
```
| cos(θ)  -sin(θ)  0 |
| sin(θ)   cos(θ)  0 |
|   0        0     1 |
```
Існують схожі матриці для обертання навколо осей X та Y.

`Новий_Вектор_Кінцівки = Матриця_Обертання * Старий_Вектор_Кінцівки`

Хоча повні обчислення є складними, це вводить ідею, що множення матриць є фундаментальним для обертання об'єктів у 3D-просторі. Додаток може використовувати це для:
*   Обчислення точних кутів суглобів користувача.
*   Надання рекомендацій на кшталт "поверніть плече назовні".
*   Порівняння орієнтації кінцівок користувача з еталонною позою, що є більш надійним, ніж просто порівняння координат (x, y, z).

Показуючи, як `A * B` з лекції можна використовувати для обертання віртуальної руки, ми надаємо потужну, відчутну причину для студентів оволодіти цією концепцією.
