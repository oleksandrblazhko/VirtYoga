# Лекція 11: Границя функції та нескінченно малі величини

**Назва лекції:** «Границя функції. Нескінченно малі та нескінченно великі функції. Визначення та основні теореми. Зв’язок між функцією, її границею та нескінченно малою».

**Назва практичного заняття:** «Знаходження границі функції».

**Пов'язаний розділ з `Docs\Lecture_1_4.txt`:** 7.3 "Границя функції неперервного аргументу".

---

### Обґрунтування користі для IT-проєкту

Перехід від дискретних послідовностей (кадрів) до **границі функції** дозволяє моделювати рух як ідеальний неперервний процес в часі `p(t)`. Це спрощує математичний опис і дає змогу застосовувати більш потужні інструменти аналізу.
-   **Ідеалізація руху:** Поняття границі функції дозволяє визначити, до якого ідеального стану `P_target` прямує положення тіла користувача `p(t)` з плином часу `t→∞`.
-   **Формалізація помилки:** Поняття **нескінченно малої функції** (н/м) є ідеальним математичним інструментом для опису помилки. Мета користувача — зробити так, щоб функція помилки `error(t) = |p(t) - P_target|` стала нескінченно малою, тобто `lim error(t) = 0`.
-   **Критерій успіху:** Критерій існування скінченної границі (Теорема 7.3.1: `f(x) = A + α(x)`) дає чіткий програмний критерій: поза досягнута, якщо поточне положення дорівнює цільовому положенню плюс нескінченно мала помилка, якою можна знехтувати.

### Зв'язок математичної формалізації із описом IT-проєкту

| Елемент опису IT-проєкту | Математична формалізація (з розділу 7.3) |
| :--- | :--- |
| Прагнення до ідеальної пози з часом | **Границя функції в точці**. `lim p(t) = P_target` при `t→∞`. |
| Похибка виконання пози | **Нескінченно мала функція `α(t)`**. `α(t) = p(t) - P_target`. |
| Досягнення цільової пози | `lim α(t) = 0` при `t→∞`. |
| Перевірка, чи поза досягнута | Критерій з теореми 7.3.1: `p(t) = P_target + α(t)`, де `α(t)` — н/м. Програмно: перевірка, чи `|p(t) - P_target| < ε`. |
| Неможливість виконати позу (напр., через втому) | **Границя функції не існує** або не дорівнює цільовому значенню. |

### Приклади математичного опису IT-проєкту

**1. Математичне визначення "Ідеальної пози":**

-   **Завдання:** Користувач входить у позу "Дерево" і намагається її утримати.
-   Його центр ваги `C(t)` рухається в часі. Ідеальне положення - `C_target`.
-   Програма аналізує `lim C(t)` при `t→∞`. Якщо ця границя існує і дорівнює `C_target`, вправа виконується правильно.
-   **Критерій "мовою н/м":** Програма обчислює функцію помилки `α(t) = C(t) - C_target`. Якщо `α(t)` є нескінченно малою (на практиці `|α(t)| < ε` протягом певного часу), то поза зараховується. Це пряме застосування теореми 7.3.1.

**2. Аналіз односторонніх границь для визначення напрямку помилки:**

-   **Завдання:** Поставити руку на певну висоту `h_target`.
-   Програма аналізує висоту руки `h(t)` в околі моменту часу `t₀`, коли користувач вважає, що виконав вправу.
-   **Аналіз:**
    -   **Лівостороння границя:** `lim h(t)` при `t→t₀-0` показує, звідки рука наближалася до цілі.
    -   **Правостороння границя:** `lim h(t)` при `t→t₀+0` показує, куди рука рушила одразу після моменту `t₀`.
-   **Сценарій:** Якщо `lim h(t) < h_target` при `t→t₀-0` і `lim h(t) > h_target` при `t→t₀+0`, це означає, що користувач не зафіксував руку, а просто "промахнувся", пройшовши крізь цільове положення. Хоча в якийсь момент він ідеально влучив у ціль (за теоремою Больцано-Коші), поза не була утримана.
-   **Рекомендація:** "Ви не зафіксували положення. Спробуйте зупинити руку точно на цільовій висоті".

**3. Нескінченно великі функції для детектування "зриву" пози:**

-   Користувач стоїть у складній балансовій позі.
-   Програма відстежує кутову швидкість `ω(t)` його коливань.
-   Поки він стоїть добре, `ω(t)` є малою величиною.
-   В момент втрати балансу `t_break` користувач починає різко падати. Швидкість `ω(t)` стрімко зростає.
-   **Аналіз:** Функція `ω(t)` в точці `t_break` поводиться як **нескінченно велика функція** (`lim ω(t) = ∞` при `t→t_break`).
-   **Дія:** Програма, зафіксувавши таке стрімке зростання, може миттєво припинити вправу і дати команду: "Обережно, ви втрачаєте баланс!".
