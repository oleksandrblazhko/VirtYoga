# Лекція 13: Еквівалентні нескінченно малі для оптимізації обчислень

**Назва лекції:** «Еквівалентні нескінченно малі функції. Порівняння нескінченно малих. Основні теореми про нескінченно малі функції. Застосування нескінченно малих функцій».

**Назва практичного заняття:** «Застосування еквівалентних нескінченно малих функцій для знаходження границі».

**Пов'язаний розділ з `Docs\Lecture_1_4.txt`:** 7.4 "Порівняння н/м, застосування еквівалентних н/м до обчислення границь".

---

### Обґрунтування користі для IT-проєкту

Поняття **еквівалентних нескінченно малих (н/м)** є потужним інструментом для оптимізації та спрощення обчислень. У контексті проєкту "WebcamPhysExercise", де аналіз пози та надання зворотного зв'язку мають відбуватися в реальному часі, швидкість обчислень є критично важливою.
-   **Прискорення обчислень:** Заміна складних функцій (тригонометричних, логарифмічних) на їхні еквівалентні поліноміальні аналоги (наприклад, `sin(x) ~ x` або `ln(1+x) ~ x` при `x→0`) дозволяє уникнути повільних обчислень. Це особливо актуально при аналізі малих відхилень від ідеальної пози, де помилка є нескінченно малою величиною.
-   **Спрощення моделей:** Використання еквівалентних н/м дозволяє лінеаризувати моделі руху. Замість того, щоб працювати зі складною нелінійною геометрією скелета, для малих рухів можна використовувати прості лінійні наближення, що значно спрощує алгоритми корекції.
-   **Аналіз порядку малості:** Порівняння нескінченно малих дозволяє визначити, які помилки є більш значущими. Наприклад, якщо відхилення руки по одній осі є `O(α)`, а по іншій — `O(α²)`, то програма розуміє, що перше відхилення є значно важливішим і потребує негайної корекції.

### Зв'язок математичної формалізації із описом IT-проєкту

| Елемент опису IT-проєкту | Математична формалізація (з розділу 7.4) |
| :--- | :--- |
| Мале кутове відхилення `α` від цільової пози | Аргумент `α` є **нескінченно малою функцією** (`α→0`). |
| Обчислення зсуву кінцівки через малий кут | Заміна `sin(α)` на еквівалентну н/м `α`. `Δx = L*sin(α) ≈ L*α`. |
| Обчислення "осідання" кінцівки під дією малого кута | Заміна `1-cos(α)` на еквівалентну н/м `α²/2`. `Δy = L*(1-cos(α)) ≈ L*α²/2`. |
| Пріоритезація корекцій | **Порівняння нескінченно малих**. Помилка `L*α` має перший порядок малості, а `L*α²/2` — другий. Отже, перша помилка є більш значущою. |

---

### Приклади математичного опису IT-проєкту

**1. Лінеаризація руху для швидкої корекції:**

-   **Завдання:** Користувач майже ідеально тримає пряму руку горизонтально, але є невелике відхилення вниз на кут `α`, де `α` - нескінченно мала величина. Програмі потрібно дати підказку: "Підніміть кисть на `Δy` сантиметрів".
-   **Точна формула:** `Δy = L * (1 - cos(α))`, де `L` - довжина руки.
-   **Проблема:** Обчислювати `cos(α)` для кожного кадру може бути затратно, особливо якщо таких обчислень багато для всіх суглобів.
-   **Оптимізація з еквівалентними н/м:** З таблиці еквівалентних нескінченно малих відомо, що `1 - cos(α) ~ α²/2` при `α→0`.
-   **Спрощена формула:** `Δy ≈ L * α²/2`.
-   **Ще простіше:** Бічне зміщення `Δx` в цьому випадку дорівнює `L * sin(α)`. Оскільки `sin(α) ~ α`, то `Δx ≈ L * α`. Це **лінійна** залежність.
-   **Результат:** Програма для фінального "доведення" пози використовує надшвидкі лінійні апроксимації, що робить її інтерфейс дуже чутливим і плавним.

**2. Пріоритезація інструкцій:**

-   Уявімо, що програма виявила два відхилення у плечовому суглобі: невеликий поворот вперед на кут `α` і невеликий поворот всередину на кут `β`. Обидва кути є н/м.
-   Зміна координат кисті внаслідок повороту `α` може бути пропорційна `α` (перший порядок малості).
-   Зміна координат кисті внаслідок повороту `β` (наприклад, якщо це обертання навколо осі руки) може бути пропорційна `β²` (другий порядок малості).
-   **Аналіз:** Оскільки `α` і `β` малі, величина `α` значно більша за `β²`. Програма, порівнявши порядки малості помилок, робить висновок, що перше відхилення є домінуючим.
-   **Рекомендація:** Програма не перевантажує користувача двома командами, а дає одну, найважливішу: "Трохи відведіть руку назад", ігноруючи менш значущу помилку другого порядку малості.

**3. Спрощення моделі фізики:**

-   У розширеній версії програма може моделювати маятникові коливання рук для динамічних вправ. Рівняння руху маятника `L*θ'' + g*sin(θ) = 0` є складним.
-   Однак для малих коливань (малі кути `θ`), ми можемо використати еквівалентність `sin(θ) ~ θ`.
-   Рівняння перетворюється на `L*θ'' + g*θ = 0`. Це рівняння **лінійного гармонічного осцилятора**, яке розв'язується набагато простіше.
-   **Результат:** Програма може аналізувати і прогнозувати малі коливання тіла, використовуючи значно простіші та швидші обчислювальні моделі.
