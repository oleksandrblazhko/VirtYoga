### Обґрунтування користі для IT-проєкту

У проєкті "WebcamPhysExercise" основною задачею є аналіз та порівняння поз людини. Бібліотека Mediapipe надає координати (x, y, z) для кожного суглоба скелета. Ці дані можна ефективно представити та обробляти за допомогою матриць. Вивчення матриць дозволить студентам:
-   Структуровано зберігати дані про положення тіла.
-   Виконувати математичні операції для порівняння поточної пози з еталонною.
-   Обчислювати необхідні корекції рухів, використовуючи лінійні перетворення.

Це робить абстрактні математичні поняття, описані в розділі 1.1, безпосередньо застосовними до розробки ключового функціоналу їхнього IT-проєкту.

### Зв'язок математичної формалізації із описом IT-проєкту

| Елемент опису IT-проєкту | Математична формалізація (з розділу 1.1) |
| :--- | :--- |
| Скелет людини, розпізнаний вебкамерою | **Матриця координат суглобів**. Якщо ми відслідковуємо N суглобів, то поточна поза людини може бути представлена як матриця `P_current` розміром N x 3. |
| Опис правильної Йога-вправи | **Еталонна матриця `P_target`** такої ж розмірності (N x 3), що містить ідеальні координати суглобів для цієї вправи. |
| Рекомендації щодо зміни положення | **Матриця різниць `D`**. Обчислюється як `D = P_target - P_current` (операція віднімання матриць). Елементи цієї матриці показують, на скільки потрібно змістити кожен суглоб по кожній осі. |
| Рух частин тіла (наприклад, обертання) | **Матричні перетворення**. Обертання кінцівки навколо суглоба описується **множенням матриці** координат відповідних точок на **матрицю обертання**. |

### Приклади математичного опису IT-проєкту

**1. Представлення пози у вигляді матриці:**

Припустимо, ми відслідковуємо 4 ключові суглоби: ліве плече (LS), лівий лікоть (LE), праве плече (RS), правий лікоть (RE). Поточна поза студента може бути матрицею `P_current` розміром 4x3:

```
      x      y      z
LS [ 0.5,  -0.2,   0.1 ]
LE [ 0.7,  -0.5,   0.2 ]
RS [ -0.5, -0.2,   0.1 ]
RE [ -0.7, -0.5,   0.2 ]
```

**2. Обчислення корекції (використання операцій над матрицями):**

Еталонна поза `P_target`:
```
      x      y      z
LS [ 0.9,  -0.2,   0.1 ]
LE [ 1.5,  -0.2,   0.1 ]
RS [ -0.9, -0.2,   0.1 ]
RE [ -1.5, -0.2,   0.1 ]
```
Програма обчислює різницю: `D = P_target - P_current`
```
      dx     dy     dz
LS [ 0.4,   0.0,   0.0 ]   // Рухати ліве плече вправо
LE [ 0.8,   0.3,  -0.1 ]   // Рухати лівий лікоть вправо та вгору
RS [ -0.4,  0.0,   0.0 ]   // Рухати праве плече вліво
RE [ -0.8,  0.3,  -0.1 ]   // Рухати правий лікоть вліво та вгору
```
Ці значення `dx, dy, dz` є основою для підказок.

**3. Розв'язання системи лінійних рівнянь матричним способом:**

Щоб знайти кут повороту для корекції, потрібно розв'язати матричне рівняння `R * V_current = V_target`, де `R` - невідома матриця обертання. Це можна зробити, знайшовши обернену матрицю `R = V_target * V_current⁻¹`. Це є прямим застосуванням теми лекції про **обернену матрицю** та **розв’язання систем лінійних рівнянь матричним способом**.
