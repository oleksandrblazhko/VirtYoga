# Лекція 25: Ряди Фур'є для гармонійного аналізу рухів

**Назва лекції:** «Ряди Фур’є. Гармонійний аналіз. Характер збіжності рядів Фур’є...».

**Назва практичного заняття:** «Розкладання парних і непарних функцій в ряди Фур’є».

**Пов'язаний розділ з `Docs\Lecture_1_12.txt`:** 15.6 "Ряди Фур’є".

---

### Обґрунтування користі для IT-проєкту

**Ряди Фур'є** та **гармонійний аналіз** є надзвичайно потужними інструментами для аналізу будь-яких періодичних або циклічних процесів, якими є багато фізичних вправ. Цей метод дозволяє розкласти складний рух на суму простих синусоїдальних коливань (гармонік).
-   **Аналіз якості руху:** Плавний, контрольований рух має простий "спектр" — основна енергія зосереджена в низьких частотах (основний рух). Навпаки, різкий, тремтячий, неконтрольований рух має значну енергію у високих частотах (дрібні ривки, тремтіння). Аналізуючи спектр Фур'є, програма може дати об'єктивну кількісну оцінку "плавності" та "контрольованості" руху.
-   **Порівняння з еталоном:** Програма може мати збережений "спектр ідеального руху", виконаного експертом. Користувач може намагатися повторити вправу так, щоб спектр його руху наблизився до еталонного.
-   **Фільтрація шумів:** Використовуючи перетворення Фур'є, можна легко відфільтрувати високочастотний "шум" (тремтіння камери, міоклонічні скорочення м'язів), залишивши для аналізу лише основний, усвідомлений рух.

### Зв'язок математичної формалізації із описом IT-проєкту

| Елемент опису IT-проєкту | Математична формалізація (з розділу 15.6) |
| :--- | :--- |
| Циклічний рух (напр. обертання рукою) | Періодична функція `p(t)`, що описує координату суглоба. |
| Розкладання руху на прості коливання | **Ряд Фур'є** для функції `p(t)`. |
| Плавність руху | **Спектр Фур'є**. Переважання низькочастотних гармонік. |
| Тремтіння, ривки | **Спектр Фур'є**. Наявність значних високочастотних гармонік. |
| Фільтрація тремтіння камери | Застосування фільтра низьких частот до спектру Фур'є сигналу. |

### Приклади математичного опису IT-проєкту

**1. Оцінка плавності кругового руху:**

-   **Вправа:** "Плавно обертайте рукою по колу".
-   Програма записує, наприклад, `x`-координату кисті `x(t)` протягом кількох обертів.
-   **Аналіз:**
    1.  Програма застосовує до `x(t)` **швидке перетворення Фур'є (FFT)** — це ефективний алгоритм для обчислення коефіцієнтів ряду Фур'є.
    2.  В результаті програма отримує спектр: амплітуди для кожної частоти (гармоніки).
    3.  Для ідеального плавного руху майже вся енергія буде на одній, основній частоті (частоті обертання).
    4.  Якщо користувач рухається нерівномірно, з'являються значні амплітуди на вищих, кратних частотах. Якщо рука тремтить, з'явиться "шум" у високочастотній частині спектру.
-   **Рекомендація:** Програма може обчислити відношення енергії високих частот до енергії основної частоти. "Ваш показник плавності: 85/100. Намагайтеся робити рух більш рівномірним, щоб зменшити кількість паразитних коливань".

**2. Порівняння з "майстром":**

-   **Калібрування:** Тренер (майстер йоги) виконує вправу, наприклад, "кішка-корова" (циклічний прогин та округлення спини). Програма записує рух точки на спині `y(t)` і зберігає її "еталонний спектр Фур'є" `S_master`.
-   **Виконання:** Користувач виконує ту ж вправу, програма будує спектр його руху `S_user`.
-   **Аналіз:** Програма порівнює `S_user` та `S_master`.
    -   Чи збігається основна частота? (Темп виконання).
    -   Чи схожі амплітуди перших кількох гармонік? (Загальна форма руху).
    -   Чи є у користувача значні піки на частотах, яких немає у майстра? (Свідчення неправильної техніки або "зайвих" рухів).
-   **Рекомендація:** "Ви виконуєте вправу в правильному темпі, але ваш рух занадто різкий у верхній точці. Спробуйте зробити перехід більш плавним, як на відео-прикладі".

**3. Видалення тремтіння м'язів:**

-   **Завдання:** Користувач тримає важку статичну позу. Його м'язи починають втомлюватися і з'являється високочастотне тремтіння.
-   **Проблема:** Це тремтіння може бути помилково інтерпретоване програмою як спроба руху, що призведе до некоректних підказок.
-   **Рішення:**
    1.  Програма отримує "зашумлений" сигнал `p(t)`.
    2.  Робить перетворення Фур'є, отримуючи спектр `S(f)`.
    3.  "Обнуляє" або сильно зменшує амплітуди всіх частот вище певного порогу (наприклад, вище 5 Гц, оскільки усвідомлені рухи зазвичай повільніші).
    4.  Робить зворотне перетворення Фур'є.
-   **Результат:** Програма отримує "очищений" сигнал, який представляє тільки плавний, основний рух (або його відсутність), і використовує вже його для аналізу пози, ігноруючи тремтіння.
