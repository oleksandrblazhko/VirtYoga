# Лекція 26: Функції багатьох змінних для комплексного аналізу пози

**Назва лекції:** «Функції багатьох змінних. Частинні похідні і диференціали вищих порядків. Теорема про рівність мішаних похідних. Застосування диференціала до наближених обчислень. Рівняння дотичної площини та нормалі до поверхні».

**Назва практичного заняття:** «Частинні похідні вищих порядків. Застосування диференціала функції багатьох змінних».

**Пов'язаний розділ з `Docs\Lecture_1_10.txt`:** 13.1 "Вступ до диференціального числення ФКЗ" та 13.2 "Частинні похідні (ч/п) і диференціали, повний диференціал".

### Обґрунтування користі для IT-проєкту

Людське тіло є складною системою, стан якої описується багатьма параметрами (кутами суглобів, положеннями частин тіла). **Функції багатьох змінних (ФКЗ)** — це природний математичний апарат для моделювання таких систем.
-   **Комплексна оцінка пози:** Замість аналізу кожного суглоба окремо, програма може визначити одну "функцію якості" `Q(θ₁, θ₂, ..., θ_N)`, яка залежить від усіх `N` кутів суглобів. Це дозволяє дати цілісну оцінку.
-   **Цілеспрямовані підказки:** **Частинні похідні** `∂Q/∂θᵢ` показують, як зміниться загальна якість пози, якщо змінити *лише один* кут `θᵢ`, утримуючи інші сталими. Це дозволяє програмі надавати дуже точні та локалізовані рекомендації: "Збільшіть кут у правому коліні, щоб покращити баланс".
-   **Аналіз взаємодії рухів:** **Мішані похідні** (теорема 13.2.4) дозволяють оцінити, як зміна одного кута впливає на ефективність зміни іншого. Наприклад, наскільки ефективніше буде підняття руки, якщо одночасно розвернути корпус.
-   **Прогнозування та згладжування:** **Повний диференціал** ФКЗ (формула 13.2.10) є основою для наближених обчислень (формула 13.2.16). Це дозволяє програмі прогнозувати, як невелика, одночасна зміна всіх кутів вплине на загальний результат, що критично для реального часу.

### Зв'язок математичної формалізації із описом IT-проєкту

| Елемент опису IT-проєкту | Математична формалізація (з розділів 13.1, 13.2) |
| :--- | :--- |
| Оцінка якості пози (або помилки) | **Функція багатьох змінних** `Score = f(θ₁, θ₂, ..., θ_N)`. |
| Як змінити один суглоб для покращення пози? | **Частинна похідна** `∂Score/∂θᵢ`. |
| Як змінити всі суглоби для покращення пози? | **Повний диференціал** `dScore = Σ(∂Score/∂θᵢ)dθᵢ`. |
| Взаємозв'язок між рухами суглобів | **Мішані похідні** `∂²Score/∂θᵢ∂θⱼ`. |
| Наближене прогнозування ефекту від малої корекції | **Наближене обчислення** `ΔScore ≈ dScore`. |

### Приклади математичного опису IT-проєкту

**1. Надання цілеспрямованих підказок:**

-   **Завдання:** Користувач виконує позу, що оцінюється "функцією відповідності" `F(θ_hips, θ_knees, θ_shoulders, ...)`.
-   **Аналіз:** Програма обчислює часткові похідні `∂F/∂θ_hips`, `∂F/∂θ_knees`, `∂F/∂θ_shoulders`.
-   Якщо `∂F/∂θ_hips > 0`, а `∂F/∂θ_knees < 0`, то:
-   **Рекомендація:** "Підніміть стегна, але не змінюйте кут у колінах". Це дуже точна і корисна порада, отримана безпосередньо з часткових похідних.

**2. Оцінка "чутливості" пози (Мішані похідні):**

-   **Завдання:** Визначити, наскільки поворот плечей впливає на кут, необхідний для випрямлення спини.
-   **Модель:** Нехай `E(θ_shoulders, θ_back)` — функція помилки, де `θ_shoulders` — кут повороту плечей, `θ_back` — кут прогину спини.
-   **Аналіз:** Програма обчислює мішану похідну `∂²E/∂θ_shoulders∂θ_back`. Якщо це значення велике, то незначна зміна кута плечей сильно впливає на прогин спини.
-   **Рекомендація:** Це дозволяє програмі ідентифікувати "чутливі" зони та надати рекомендації: "Коли ви повертаєте плечі, будьте уважні до положення спини, воно сильно змінюється".

**3. Прогнозування результату малої корекції (Повний диференціал):**

-   **Завдання:** Користувач майже в ідеальній позі. Програма хоче спрогнозувати, як невелика корекція в декількох суглобах одночасно вплине на "функцію помилки" `E(θ₁, θ₂, ...)`.
-   **Аналіз:** Замість повного перерахунку `E` для нових значень, програма використовує наближення за допомогою повного диференціала (формула 13.2.16):
    `ΔE ≈ dE = (∂E/∂θ₁)Δθ₁ + (∂E/∂θ₂)Δθ₂ + ...`
-   **Застосування:** Це дозволяє програмі швидко моделювати різні сценарії "що якщо?" і пропонувати користувачеві комбінації малих рухів, які дадуть найкращий результат. Наприклад, "Якщо ви трохи опустите праве плече і трохи піднімете ліве стегно, ваша помилка зменшиться на 5%".
