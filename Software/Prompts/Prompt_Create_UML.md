# Person
Ти експерт із моделювання предметної області з використанням UML-діаграм.

# Tasks:
1) прочитай файл із описом йога-вправ;
2) для різних описів поз порівняй розділи "Загальний опис", "Позитивні впливи", "Покрокові інструкції", 
"Модифікації", "Поширені помилки", "Безпека та запобіжні заходи" та визнач їх об'єднуючі властивості:
- сутності предметної області;
- властивості сутностей;
3) створи UML-діаграм концептуальних класів, які описують сутності

# Context:
1) визначити імена класів, аналізуючи загальні іменники із опису сутностів;
2) визначити імена атрибутів класів, аналізуючи властивості сутностей
3) враховувати відмінність більш загальних концептуальних класів від більш конкретних програмних класів:
    - не описувати типи атрибутів;
    - атрибути повині бути атомарними, а не вказувати на складну структуру, наприклад, списки;
4) визначити зв'язки між класами:
    - узагальнення (спадкування) через аналіз можливих ієрархічних зв’язків між класами;
    - іменована асоціація через аналіз дієслів в описах властивостей
    - агрегована асоціація через аналіз слів у зв’язках між класами,
    коли взаємозв’язок між класами включає слова "містить" або "є частиною";
    - для зв’язків іменованих асоціацій та агрегатних асоціацій 
    визначити кратність типу «one-to-one», «one-to-many»;

# Format:
1) опис йога-вправ розміщено у файлі YogaDocs\YogaPoses.md;
2) опиши UML-діаграму концептуальних класів мовою розмітки тексту Mermaid;
3) надай українські коментарі для назв класів та атрибутів класів;
4) збережи UML-діаграму концептуальних класів у файлі Software\Docs\UML.md
5) на початку файлу із рішенням вкажи назву програму, яка виконала завдання, наприклад, Quen CLI, Gemini CLI;
6) використовуй приклад створення UML-діаграму концептуальних класів:
